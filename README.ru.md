**G-Labs Voice Studio** — это настольное приложение для многоязычного синтеза речи: клонируйте голос по короткому образцу, проектируйте новые голоса по характеристикам, пишите многоголосые диалоги и преобразуйте речь в текст. Тёмный интерфейс, модели работают локально на вашей машине — ни аудио, ни текст не покидают устройство после загрузки модели.

## Возможности

- **🔊 Клонирование голоса** — дайте образец длиной 5–10 секунд, и приложение прочитает любой текст этим голосом.
- **🎛️ Текст в речь** — опишите голос по характеристикам (пол, возраст, высота, стиль, акцент), образец аудио не нужен.
- **💬 Многоголосый диалог** — пишите сценарий для нескольких персонажей с помощью синтаксиса `<Имя>: реплика` и назначайте каждому персонажу голос из библиотеки.
- **📝 Речь в текст (ASR)** — расшифровывайте аудио- и видеофайлы. Поддерживаются MP3, WAV, M4A, FLAC, MP4, MOV…
- **🗂 Библиотека голосов + 30 заводских голосов** — приложение поставляется с 30 готовыми голосами (мужские/женские, разные тембры). Вы также можете сохранять собственные. Поставьте ⭐ голосу, чтобы закрепить его наверху, и **создавайте резервные копии и восстанавливайте** их как портативные файлы `.vcp` для обмена.
- **🎚 Регулировка скорости + экспорт с учётом скорости** — меняйте скорость предпросмотра под волновой формой; экспортированный файл воспроизводится точно с этой скоростью (высота тона сохраняется).
- **📄 Экспорт субтитров (SRT)** — при экспорте в один объединённый файл рядом автоматически создаётся `.srt`, готовый для видеомонтажа.
- **🔤 Личный фонетический словарь** — встретились `100%`, `25°C`, `m²`? Введите произношение один раз, и приложение запомнит его на следующий раз (отдельно по каждому языку вывода).
- **✨ ИИ-подсказка образца текста** — Whisper автоматически расшифрует образец аудио, чтобы не пришлось перепечатывать вручную (всё равно вычитайте перед генерацией).
- **⬇ Скачивание по строкам** — у каждой строки таблицы сценария есть своя кнопка скачивания — не нужно ждать всей партии.
- **Поддержка 600+ языков** — говорите более чем на 600 языках (вьетнамский, английский, китайский, японский, корейский, французский, немецкий, испанский и многие языки малых народов).
- **Пакетная обработка** — импортируйте сценарии `.txt` или `.srt`, запускайте пакетную обработку, экспортируйте аудио с сохранением временных меток.
- **Умная проверка совместимости GPU** — если ваш GPU не может запустить ускоренные ядра, приложение **автоматически переключается на многоядерный CPU** и честно сообщает об этом, вместо того чтобы падать с ошибкой CUDA.
- **Интерфейс на 9 языках** — Tiếng Việt, English, Português, Türkçe, 简体中文, हिन्दी, বাংলা, اردو, Русский.

## Установка

| Платформа | Файл | Размер |
|---|---|---|
| Windows x64 | `GLabsVoiceStudio-v1.0.8-win.zip` | ~3 GB |
| macOS Apple Silicon | `GLabsVoiceStudio-v1.0.8-arm64.dmg` | ~2 GB |

### Windows (портативно, установка не требуется)

1. Скачайте `GLabsVoiceStudio-v1.0.8-win.zip`.
2. Распакуйте в любую папку (на диске нужно минимум 10 GB свободного места).
3. Откройте распакованную папку и дважды кликните по `GLabsVoiceStudio.exe`.

> Хотите ярлык на рабочем столе? Правый клик по `GLabsVoiceStudio.exe` → *Отправить* → *Рабочий стол (создать ярлык)*.

> **⏳ Первый запуск может занять 30–60 секунд (заставка зависнет около ~90%) — пожалуйста, подождите и не закрывайте.** Windows должен просканировать приложение и файлы GPU (автоматическая мера безопасности, медленно только в первый раз). Со второго запуска приложение открывается быстро.

### 🍎 macOS Apple Silicon

1. Скачайте **`GLabsVoiceStudio-v1.0.8-arm64.dmg`** из официального источника.
2. Дважды кликните по файлу `.dmg`, чтобы открыть его.
3. Перетащите иконку **G-Labs Voice Studio** в папку **Applications**.
4. Откройте **Applications**, **правый клик** по **G-Labs Voice Studio** → выберите **Open**.

> ⚠️ **Первый запуск:** macOS может показать сообщение *«Программа от неизвестного разработчика»*. Правый клик по приложению → **Open** → нажмите **Open** в диалоговом окне. Нужно только один раз.

> 📁 **Файлы вывода** (аудио, сценарии) по умолчанию сохраняются в `~/Documents/G-Labs Voice Studio/output/`.

> **⏳ Первый запуск может занять 30–60 секунд (заставка зависнет около ~90%) — пожалуйста, подождите и не закрывайте.** macOS выполняет полную проверку безопасности при первом запуске. Со второго запуска приложение открывается быстро.

#### 🍎 Исправление ошибки *«App is damaged»* на macOS

Когда вы скачиваете `.dmg` из интернета, macOS автоматически прикрепляет к файлу **карантинный флаг**. Если приложение не нотаризовано Apple, Gatekeeper заблокирует его и может сообщить *«Программа повреждена и не может быть открыта»*.

**Способ 1: правый клик → Open** *(рекомендуется)*

1. Откройте **Applications**.
2. **Правый клик** по *G-Labs Voice Studio* → **Open**.
3. Нажмите **Open** ещё раз в окне предупреждения.

**Способ 2: использовать Terminal** (если способ 1 всё ещё показывает ошибку)

Откройте **Terminal** (Launchpad → Other → Terminal), вставьте следующую команду и нажмите Enter:

```bash
xattr -cr "/Applications/G-Labs Voice Studio.app"
```

Это очищает все расширенные атрибуты (включая карантинный флаг) с бандла приложения. После выполнения откройте приложение обычным способом. Это нужно сделать **один раз** на каждую скачанную версию.

**Способ 3: System Settings → Privacy & Security**

1. Попробуйте открыть приложение обычным способом (оно будет заблокировано).
2. Откройте **System Settings** → **Privacy & Security**.
3. Прокрутите вниз, найдите *«G-Labs Voice Studio was blocked…»* → нажмите **Open Anyway**.
4. Подтвердите с помощью Touch ID или пароля администратора.

## С чего начать

### Часть 1 — первый запуск (разовая настройка)

1. Откройте приложение — приветственный экран предложит 9 языков интерфейса. Выберите нужный.
2. **Войдите в систему** — нажмите значок ⚙️ в левой боковой панели → вкладка *License* → **«Sign in with Google»**.
3. **Скачайте ИИ-модель** — переключитесь на вкладку *Environment* (или дождитесь подсказки приложения) → нажмите **«Download model»**. Несколько GB, дождитесь окончания.
4. Закройте настройки.

> 💡 Изменить язык интерфейса позже: Settings → Language.

### Часть 2 — Клонирование голоса 🔊

Клонируйте голос из образца записи.

1. Откройте вкладку **🔊 Клонирование голоса**.
2. Выберите **язык вывода** в верхней части вкладки (например, вьетнамский, английский…).
3. В поле *Образец аудио* нажмите **«Выбрать...»** и выберите образец (5–10 секунд, чистый голос, мало фонового шума).
4. **Обязательно:** введите *Образец текста* — точную транскрипцию образца аудио (с правильной пунктуацией и без опечаток).
    > 💡 Нажмите **«✨ Подсказка ИИ»** и Whisper расшифрует образец за вас — всё равно вычитайте перед генерацией.
5. Вставьте целевой текст в *Содержимое текста* (или нажмите **«📂 Импорт (.txt, .srt)»**, чтобы загрузить из `.txt` / `.srt`).
6. Нажмите **«📋 Добавить в таблицу»** — приложение разделит ваш текст на отдельные предложения.
7. (Необязательно) Нажмите **🔤 Произношение**, чтобы просмотреть фонетический словарь — если ваш текст содержит специальные символы (например, `100%`), приложение спросит, как их читать.
8. Нажмите **«▶ Начать обработку»** → появится диалог подтверждения для проверки транскрипции образца → подтвердите для запуска.
9. По окончании нажмите **«💾 Экспорт аудио»** (один объединённый файл с сопроводительным `.srt`) или используйте **⬇** в строке, чтобы скачать только её.

### Часть 3 — Текст в речь 🎛️

Создайте новый голос по описанию характеристик — образец не нужен.

1. Откройте вкладку **🎛️ Текст в речь**.
2. Выберите **язык вывода**.
3. Откройте панель **Дизайн голоса** и выберите: *Пол*, *Возраст*, *Высота*, *Стиль*, *Акцент*.
    > 💡 Хотите повторно использовать сохранённый голос из **Библиотеки голосов**? Выберите его прямо из выпадающего списка — приложение пропустит прогрев первой строки и начнёт сразу с первой.
4. Вставьте ваш текст и нажмите **«📋 Добавить в таблицу»**.
5. Нажмите **«▶ Начать обработку»**.
6. После генерации: кликните по строке таблицы, которую хотите сохранить → нажмите **«💾 Сохранить»** на панели *Библиотека голосов*, чтобы повторно использовать его позже.
7. Нажмите **«💾 Экспорт аудио»**, чтобы сохранить результат.

### Часть 4 — Многоголосый диалог 💬 *(новое)*

Генерируйте аудио с многоперсонажным диалогом, один голос на каждого говорящего — идеально для подкастов, аудиоспектаклей и видео в формате интервью.

1. Откройте вкладку **💬 Многоголосый диалог**.
2. Выберите **язык вывода**.
3. Нажмите **📋 Пример диалога** (вверху справа от текстового поля), чтобы увидеть рабочий пример — приложение вставит образец и откроет редактор подробностей.
4. Пишите ваш сценарий с использованием такого синтаксиса:
    ```
    <Host>: Welcome everyone, today we have a very special conversation lined up.
    <Anna>: Hi, I'm really glad to be part of the show today.
    <Ben>: Same here, thanks for having us on. What's the topic?
    ```
    > 💡 Имя персонажа указывается внутри `< >`, затем `:` и реплика. Двоеточие необязательно — `<Anna> Hi there` тоже работает. Регистр имён не учитывается.
5. Нажмите **«🎭 Разобрать диалог»** — приложение разделит сценарий на строки и покажет колонку «Персонаж».
6. Панель **Назначение голосов** откроется автоматически — выберите голос из библиотеки для каждого персонажа.
7. (Необязательно — *новое*) У каждой строки персонажа есть собственный ползунок **Скорость:** (0,5x → 2x) прямо под выбором голоса. Персонажи могут работать в независимом темпе в одной партии — например, ведущий говорит медленно, гость быстрее.
8. Нажмите **«▶ Начать обработку»** — каждая реплика будет прочитана голосом назначенного персонажа.
9. Нажмите **«💾 Экспорт аудио»**, чтобы сохранить результат (соответствующий `.srt` включён).

### Часть 5 — Речь в текст 📝

Расшифруйте речь из имеющегося аудио- или видеофайла.

1. Откройте вкладку **📝 Речь в текст**.
2. Нажмите **«Выбрать...»** и выберите аудио- или видеофайл (MP3, WAV, M4A, FLAC, MP4, MOV…).
3. Нажмите **«▶ Начать извлечение»** — приложение проанализирует аудио, разобьёт его на речевые сегменты и расшифрует каждый.
4. Результаты появятся в таблице с временными метками по предложениям. Их можно редактировать прямо там.
5. Нажмите **«💾 Экспортировать результат»**, чтобы сохранить как `.txt` (обычный текст) или `.srt` (с временными метками, готово для субтитров).

## 💡 Советы для продвинутых

### Мастеринг звука *(новое)*
- Все три вкладки генерации (Клонирование голоса / Текст в речь / Многоголосый диалог) имеют сворачиваемый блок **Мастеринг звука** в середине формы.
- 6 встроенных пресетов:
  - 📻 **Радио** *(по умолчанию)* — стандарт радио/подкаста, тёплый и сжатый.
  - 🎬 **Кино** — просторная реверберация, мягкая компрессия, киношное качество.
  - 🎙️ **Подкаст** — близкий микрофон, сильная компрессия, без реверберации.
  - 🔇 **Чистый** — вывод модели как есть, без обработки.
  - ☀️ **Тёплый** — усиленные нижние середины, уютное ощущение.
  - ✨ **Яркий** — чёткие верха, воздушное ощущение.
- Измените пресет в одной вкладке — две другие синхронизируются автоматически. Чтобы получить точно такой же вывод, как в предыдущей версии (v1.0.4): выберите **🔇 Чистый** и снимите галочку **Выровнять громкость между фразами**.

### Автоосвобождение памяти при простое *(новое)*
- По умолчанию: если приложение бездействует **5 минут**, ИИ-модель автоматически выгружается из VRAM/RAM для освобождения системных ресурсов.
- При следующей генерации модель загрузится снова (~30–60 с).
- Перейдите на вкладку **Настройка среды** → **Автоосвобождение VRAM** для настройки таймаута (0 = отключено, 1–120 минут).

### Библиотека голосов и звезда «избранное»
- У каждого голоса в библиотеке в начале строки есть ☆. Нажмите её → она превратится в ★ → этот голос переместится наверх списка (и останется там в следующий раз).
- Нажмите ★ ещё раз, чтобы убрать из избранного.
- Все 30 заводских голосов (Achernar, Aoede, Kore, Puck…) тоже можно добавить в избранное.

### Фонетический словарь
- Когда ваш текст содержит `%`, `$`, `°C`, `m²`, названия брендов… нажмите **🔤 Произношение** перед генерацией.
- Когда приложение видит токен впервые, оно спрашивает, как его читать (например, `%` → ` процент`).
- Каждый токен нужно ввести только **один раз** — приложение запомнит его для каждого языка вывода. В следующий раз применится автоматически.
- Один и тот же токен в разных языках хранит отдельные произношения (например, вьетнамское против английского).

### Теги эмоций и невербальных звуков
- Впишите любой из этих тегов прямо в текст — голос воспроизведёт соответствующий невербальный звук. Работает в **Клонировании голоса**, **Тексте в речь** и **Многоголосом диалоге**.
- Пишите тег точно как показано, вместе с квадратными скобками — отдельно или между предложениями (напр. `Это так смешно [laughter] не могу остановиться.`).

| Тег | Звук |
|---|---|
| `[laughter]` | Смех |
| `[sigh]` | Вздох |
| `[confirmation-en]` | Согласие — «mm-hmm» |
| `[question-en]` | Вопросительная интонация |
| `[question-ah]` | Вопрос — «ah?» |
| `[question-oh]` | Вопрос — «oh?» |
| `[question-ei]` | Вопрос — «ei?» |
| `[question-yi]` | Вопрос — «yi?» |
| `[surprise-ah]` | Удивление — «ah!» |
| `[surprise-oh]` | Удивление — «oh!» |
| `[surprise-wa]` | Удивление — «wa!» |
| `[surprise-yo]` | Удивление — «yo!» |
| `[dissatisfaction-hnn]` | Недовольство — «hnn» |

> 💡 Теги воспринимаются как выразительные подсказки, а не читаются буквально. Сила эффекта зависит от языка и голоса — сначала проверьте на короткой фразе.

### Скорость воспроизведения + экспорт
- После генерации аудио в панели под волновой формой есть **выпадающий список скорости** (0,5x → 2x, по шагам).
- Измените скорость → предпросмотр в плеере обновится мгновенно.
- Экспортированный файл сохраняет точно эту скорость, а **высота тона сохраняется** (без эффекта «бурундука» — используется временная растяжка).

### Экспорт SRT
- В панели *Настройки экспорта* галочка **«Также экспортировать субтитры (.srt)»** включена по умолчанию.
- Когда вы экспортируете в один объединённый файл, приложение создаёт `name.srt` рядом с `name.wav`.
- Временные метки в SRT отражают фактическую длительность каждой строки (с учётом регулировки скорости).

## Системные требования

|   | Минимум | Рекомендуется |
|---|---|---|
| **ОС** | Windows 10 (64-bit), macOS 12 Monterey | Windows 11, macOS 13 или новее |
| **RAM** | 8 GB | 16 GB или больше |
| **VRAM (GPU)** | 4 GB (автоматически выгружает TTS на CPU) | 8 GB+ (NVIDIA RTX 3060 или лучше) |
| **Диск** | 10 GB свободно (модели + кеш) | 20 GB+ SSD |
| **GPU** | Необязательно — работает на CPU | NVIDIA CUDA · Apple Silicon (Metal) |

> 💡 **Совет:** на GPU с ≤ 8 GB VRAM приложение автоматически выгружает TTS на CPU во время расшифровки — настройка не требуется. Выделенный GPU не обязателен; весь конвейер работает на CPU (просто медленнее).

### Замечания по платформам

**Windows x64**
- **GPU NVIDIA, серия RTX 20 или новее** (вычислительная способность ≥ 7.0 — RTX 20/30/40/50, Titan V, Tesla V100…). Старые карты вроде серии GTX 10 (GTX 1060/1070/1080) **не могут запускать ускоренные ядра**; приложение определяет это и автоматически переходит на CPU.
- Драйвер NVIDIA с поддержкой CUDA 12.8.

**macOS**
- Поддерживается только **Apple Silicon** (M1/M2/M3/M4…) — используется ускорение Metal от Apple. Intel-Mac не поддерживаются.

> Машины без совместимого GPU **автоматически работают на многоядерном CPU** (приложение настраивает количество потоков PyTorch / BLAS под количество физических ядер). Примерно в 5–10 раз медленнее GPU, но всё ещё пригодно для коротких задач озвучивания.

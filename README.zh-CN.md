**G-Labs Voice Studio** 是一款多语言语音合成桌面应用：只需一段简短的样本音频即可克隆声音，按属性设计全新的声音，编写多角色对话脚本，并将语音转写为文本。深色主题界面，模型完全在本地运行——模型下载完成后，您的音频和文本数据都不会离开您的设备。

## 功能特性

- **🔊 语音克隆** —— 提供一段 5–10 秒的样本音频，应用即可用该声音朗读任意文本。
- **🎛️ 文本转语音 / 声音设计** —— 通过属性（性别、年龄、音调、风格、口音）描述一个声音，无需任何样本音频。
- **💬 多角色对话** —— 使用 `<名称>: 台词` 语法编写多角色脚本，并为每个角色分配声音库中的声音。
- **📝 语音转文本 (ASR)** —— 转写音频/视频文件。支持 MP3、WAV、M4A、FLAC、MP4、MOV 等格式。
- **🗂 声音库 + 30 个内置声音** —— 应用内置 30 个开箱即用的声音（男声/女声，多种音色）。您也可以保存自己的声音。点击 ⭐ 即可将常用声音置顶；还能**备份与恢复**为便于分享的 `.vcp` 文件。
- **🎚 速度调节 + 速度同步导出** —— 在波形下方调整试听速度，导出的文件将以完全相同的速度播放（音调保持不变）。
- **📄 字幕 (SRT) 导出** —— 以合并为单文件的方式导出时，应用会自动同时生成 `.srt` 字幕，方便视频剪辑使用。
- **🔤 个人发音词典** —— 遇到 `100%`、`25°C`、`m²` 这类字符？只需输入一次发音，应用就会记住，下次自动应用（按输出语言保存）。
- **✨ AI 样本文本建议** —— Whisper 会自动转写样本音频，您无需从头打字（生成前仍建议校对）。
- **⬇ 单行下载** —— 脚本表格中每一行都有独立的下载按钮——无需等待整个批次完成。
- **支持 600 多种语言** —— 可朗读 600 多种语言（越南语、英语、中文、日语、韩语、法语、德语、西班牙语以及众多少数民族语言）。
- **批量处理** —— 导入 `.txt` 或 `.srt` 脚本，批量运行并导出音频，同时保留时间戳。
- **智能 GPU 兼容性检测** —— 如果您的 GPU 无法运行加速内核，应用会**自动回退到多核 CPU**并明确告知您，而不是直接以 CUDA 错误崩溃。
- **9 种界面语言** —— Tiếng Việt、English、Português、Türkçe、简体中文、हिन्दी、বাংলা、اردو、Русский。

## 安装

| 平台 | 文件 | 大小 |
|---|---|---|
| Windows x64 | `GLabsVoiceStudio-v1.0.8-win.zip` | 约 3 GB |
| macOS Apple Silicon | `GLabsVoiceStudio-v1.0.8-arm64.dmg` | 约 2 GB |

### Windows（便携版，无需安装）

1. 下载 `GLabsVoiceStudio-v1.0.8-win.zip`。
2. 解压到任意文件夹（所在磁盘至少需有 10 GB 可用空间）。
3. 打开解压后的文件夹，双击 `GLabsVoiceStudio.exe`。

> 想要桌面快捷方式？右键点击 `GLabsVoiceStudio.exe` → *发送到* → *桌面快捷方式*。

> **⏳ 首次启动可能需要 30–60 秒（启动画面会在约 90% 处停顿）—— 请耐心等待，不要关闭。** Windows 需要扫描应用和 GPU 文件（自动安全检查，仅首次较慢）。从第二次启动开始，应用会快速打开。

### 🍎 macOS Apple Silicon

1. 从官方发布渠道下载 **`GLabsVoiceStudio-v1.0.8-arm64.dmg`**。
2. 双击 `.dmg` 文件打开。
3. 将 **G-Labs Voice Studio** 图标拖入 **应用程序** 文件夹。
4. 打开 **应用程序**，**右键点击** **G-Labs Voice Studio** → 选择 **打开**。

> ⚠️ **首次启动：** macOS 可能提示 *"来自身份不明开发者的应用"*。请右键点击应用 → **打开** → 在弹窗中点击 **打开**。仅需操作一次。

> 📁 **输出文件**（音频、脚本）默认保存到 `~/Documents/G-Labs Voice Studio/output/`。

> **⏳ 首次启动可能需要 30–60 秒（启动画面会在约 90% 处停顿）—— 请耐心等待，不要关闭。** macOS 首次启动时会执行完整的安全检查。从第二次启动开始，应用会快速打开。

#### 🍎 修复 macOS 上的 *"应用已损坏"* 提示

当您从互联网下载 `.dmg` 文件时，macOS 会自动给文件附加**隔离标记**。如果该应用未经 Apple 公证，Gatekeeper 会阻止运行，并可能提示 *"应用已损坏，无法打开"*。

**方法 1：右键 → 打开** *（推荐）*

1. 打开 **应用程序**。
2. **右键点击** *G-Labs Voice Studio* → **打开**。
3. 在警告对话框中再次点击 **打开**。

**方法 2：使用终端**（若方法 1 仍然报错）

打开 **终端**（Launchpad → 其他 → 终端），粘贴以下命令并按回车：

```bash
xattr -cr "/Applications/G-Labs Voice Studio.app"
```

此命令会清除应用包的所有扩展属性（包括隔离标记）。执行后即可正常打开应用。每个下载版本仅需执行**一次**。

**方法 3：系统设置 → 隐私与安全性**

1. 尝试正常打开应用（会被阻止）。
2. 打开 **系统设置** → **隐私与安全性**。
3. 滚动到底部，找到 *"已阻止使用 G-Labs Voice Studio…"* → 点击 **仍要打开**。
4. 用 Touch ID 或管理员密码确认。

## 快速上手

### 第 1 部分 —— 首次启动（一次性设置）

1. 打开应用 —— 欢迎界面会提供 9 种界面语言，选择您想要的语言。
2. **登录** —— 点击左侧栏的 ⚙️ 齿轮图标 → *授权* 标签页 → **"使用 Google 登录"**。
3. **下载 AI 模型** —— 切换到 *环境设置* 标签页（或等待应用提示）→ 点击 **"下载模型"**。约几 GB；请等待下载完成。
4. 关闭设置。

> 💡 之后可随时修改界面语言：⚙️ 环境设置 → 界面语言。

### 第 2 部分 —— 语音克隆 🔊

通过样本录音克隆一个声音。

1. 打开 **🔊 语音克隆** 标签页。
2. 在标签页顶部选择 **语言**（例如越南语、英语等）。
3. 在 *样本音频* 字段点击 **"选择..."**，选择一段样本（5–10 秒，声音清晰，背景噪音低）。
4. **必填：** 输入 *样本文本* —— 即样本音频的完整准确转写（标点正确、不要错字）。
    > 💡 点击 **"✨ AI 建议"**，Whisper 会自动转写 —— 生成前仍需校对。
5. 将目标文本粘贴到 *文本内容* 框（或点击 **"📂 导入 (.txt, .srt)"** 从 `.txt` / `.srt` 文件加载）。
6. 点击 **"📋 添加到表格"** —— 应用会将文本拆分为独立的句子。
7. （可选）点击 **🔤 发音** 查看发音词典 —— 若文本中包含特殊字符（例如 `100%`），应用会询问您应当如何朗读。
8. 点击 **"▶ 开始处理"** → 弹出确认对话框校对样本转写 → 确认后开始运行。
9. 完成后，点击 **"💾 导出音频"**（合并为单个文件并附带 `.srt`），或在某一行上点击 **⬇** 仅下载该行。

### 第 3 部分 —— 文本转语音 (TTS) 🎛️

通过属性描述创建一个全新的声音 —— 无需任何样本。

1. 打开 **🎛️ 文本转语音** 标签页。
2. 选择 **语言**。
3. 打开 **声音设计** 面板，选择：*性别*、*年龄*、*音调*、*风格*、*口音*。
    > 💡 想复用 **声音库** 中已保存的声音？直接从下拉菜单中选择即可 —— 应用会跳过首行预热，直接从第 1 行开始生成。
4. 粘贴文本并点击 **"📋 添加到表格"**。
5. 点击 **"▶ 开始处理"**。
6. 生成完成后：在表格中点击一行您想保留的内容 → 在 *声音库* 面板中点击 **"💾 保存"** 以便日后复用。
7. 点击 **"💾 导出音频"** 保存结果。

### 第 4 部分 —— 多角色对话 💬 *（新）*

生成多角色对话音频，每个角色一种声音 —— 非常适合播客、广播剧和访谈类视频。

1. 打开 **💬 多角色对话** 标签页。
2. 选择 **语言**。
3. 点击 **📋 示例对话**（位于文本框右上方），即可查看可用示例 —— 应用会粘贴示例并打开详细文本编辑窗口。
4. 按以下语法编写脚本：
    ```
    <Host>: 欢迎大家，今天我们将带来一场非常特别的对话。
    <Anna>: 嗨，我很高兴今天能参与这期节目。
    <Ben>: 我也是，感谢邀请我们。今天的话题是什么？
    ```
    > 💡 角色名放在 `< >` 内，后面跟 `:` 再接台词。冒号可省略 —— `<Anna> 嗨` 也可以。名称不区分大小写。
5. 点击 **"🎭 解析对话"** —— 应用会将脚本拆分成多行，并显示"角色"列。
6. **声音分配** 面板会自动打开 —— 为每个角色选择一个声音库中的声音。
7. （可选 —— *新*）每个角色行的声音选择器下方都有独立的 **语速** 滑块（0.5x → 2x）。同一批次中不同角色可以使用不同语速 —— 例如主持人慢一点、嘉宾快一点。
8. 点击 **"▶ 开始处理"** —— 每一行都会用所分配角色的声音朗读。
9. 点击 **"💾 导出音频"** 保存结果（会同时生成对应的 `.srt`）。

### 第 5 部分 —— 语音转文本 📝

从现有的音频/视频文件中转写语音。

1. 打开 **📝 语音转文本** 标签页。
2. 点击 **"选择..."** 并挑选一个音频/视频文件（MP3、WAV、M4A、FLAC、MP4、MOV 等）。
3. 点击 **"▶ 开始提取"** —— 应用会分析音频，按语音段切分并逐段转写。
4. 结果以表格呈现，含每句的时间戳。您可以直接在表格中编辑。
5. 点击 **"💾 导出结果"** 保存为 `.txt`（纯文本）或 `.srt`（带时间戳，可直接用作字幕）。

## 💡 进阶技巧

### 音频母带处理 *（新）*
- 三个生成标签页（🔊 语音克隆 / 🎛️ 文本转语音 / 💬 多角色对话）的表单中部都有一个可折叠的 **音频母带处理** 区域。
- 内置 6 种预设：
  - 📻 **广播** *（默认）* —— 广播/播客标准，温暖且经过压缩。
  - 🎬 **电影** —— 宽广混响，轻度压缩，电影级品质。
  - 🎙️ **播客** —— 近距离麦克风，重度压缩，无混响。
  - 🔇 **原始** —— 模型原始输出，不做任何处理。
  - ☀️ **温暖** —— 增强中低频，温暖舒适。
  - ✨ **明亮** —— 清脆高频，通透感强。
- 在任一标签页修改预设 → 其余两个会自动同步。若想得到与上个版本（v1.0.4）完全相同的输出：选择 **🔇 原始** 并取消勾选 **统一各句音量**。

### 空闲时自动释放显存 *（新）*
- 默认：应用闲置 **5 分钟** 后，会自动从显存/内存中卸载 AI 模型，释放系统资源。
- 下次生成时模型会重新加载（约 30–60 秒）。
- 在 **⚙️ 环境设置** 标签页 → **自动释放显存** 中调整超时（0 = 关闭，1–120 分钟）。

### 声音库与收藏星标
- 声音库中每个声音行首都有 ☆。点击它 → 变为 ★ → 该声音会跳到列表顶部（并在下次打开时保持置顶）。
- 再次点击 ★ 即可取消收藏。
- 全部 30 个内置声音（Achernar、Aoede、Kore、Puck 等）也可以收藏。

### 发音词典
- 当您的文本包含 `%`、`$`、`°C`、`m²`、品牌名等内容时，请在生成前点击 **🔤 发音**。
- 应用首次遇到某个符号时会询问您应当如何朗读（例如 `%` → `百分之`）。
- 每个符号只需输入**一次** —— 应用会按输出语言记住它。下次自动应用。
- 同一符号在不同语言下保留各自独立的发音（例如越南语 vs 英语）。

### 情绪与非语言标签
- 直接在输入文本中写入这些标签，语音便会发出对应的非语言声音。在**语音克隆**、**文本转语音**和**多角色对话**中均可使用。
- 严格按下方所示书写标签（保留方括号），可单独使用或夹在句子之间（例如 `太好笑了 [laughter] 我停不下来。`）。

| 标签 | 声音 |
|---|---|
| `[laughter]` | 笑声 |
| `[sigh]` | 叹气 |
| `[confirmation-en]` | 应答 —"mm-hmm" |
| `[question-en]` | 疑问语气 |
| `[question-ah]` | 疑问语气 —"ah?" |
| `[question-oh]` | 疑问语气 —"oh?" |
| `[question-ei]` | 疑问语气 —"ei?" |
| `[question-yi]` | 疑问语气 —"yi?" |
| `[surprise-ah]` | 惊讶 —"ah!" |
| `[surprise-oh]` | 惊讶 —"oh!" |
| `[surprise-wa]` | 惊讶 —"wa!" |
| `[surprise-yo]` | 惊讶 —"yo!" |
| `[dissatisfaction-hnn]` | 不满 —"hnn" |

> 💡 标签会被当作情绪提示，而不会被逐字读出。具体效果因语言和音色而异——建议先用一句短句测试。

### 播放速度 + 导出
- 音频生成后，波形下方的工具栏中有一个 **速度下拉菜单**（0.5x → 2x，按级递增）。
- 修改速度 → 播放器立即更新试听。
- 导出文件会保留这个完全相同的速度，且**音调保持不变**（不会出现"花栗鼠效果" —— 使用时间拉伸算法）。

### SRT 导出
- 在 *导出设置* 面板中，**"同时导出字幕 (.srt)"** 默认开启。
- 当以合并为单文件的方式导出时，应用会在 `name.wav` 旁创建 `name.srt`。
- SRT 中的时间戳反映的是速度调整后每句的实际时长。

## 系统要求

|   | 最低配置 | 推荐配置 |
|---|---|---|
| **操作系统** | Windows 10（64 位）、macOS 12 Monterey | Windows 11、macOS 13 或更新 |
| **RAM** | 8 GB | 16 GB 或更多 |
| **VRAM (GPU)** | 4 GB（TTS 会自动转用 CPU） | 8 GB 以上（NVIDIA RTX 3060 或更高） |
| **磁盘空间** | 10 GB 可用（模型 + 缓存） | 20 GB 以上 SSD |
| **GPU** | 可选 —— 仅 CPU 也可运行 | NVIDIA CUDA · Apple Silicon (Metal) |

> 💡 **提示：** 在 VRAM ≤ 8 GB 的 GPU 上，应用会在转写期间自动将 TTS 转用 CPU —— 无需任何配置。本程序并不强制需要独立 GPU；整套流程都可以在 CPU 上运行（只是速度较慢）。

### 平台说明

**Windows x64**
- **NVIDIA GPU，RTX 20 系列或更新**（计算能力 ≥ 7.0 —— RTX 20/30/40/50、Titan V、Tesla V100 等）。GTX 10 系列等较老显卡（GTX 1060/1070/1080）**无法运行加速内核**；应用会检测到并自动回退到 CPU。
- NVIDIA 驱动需支持 CUDA 12.8。

**macOS**
- 仅支持 **Apple Silicon**（M1/M2/M3/M4 等）—— 使用 Apple 的 Metal 加速。不支持 Intel 芯片的 Mac。

> 没有兼容 GPU 的机器会**自动以多核 CPU 运行**（应用会根据您的物理核心数调整 PyTorch / BLAS 线程数）。比 GPU 慢约 5–10 倍，但短篇配音任务仍可使用。
